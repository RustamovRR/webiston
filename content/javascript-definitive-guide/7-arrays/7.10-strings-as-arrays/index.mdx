# Satr'lar massiv sifatida

JavaScript satr'lari o‘zini UTF-16 Unicode belgilaridan iborat **faqat o‘qish uchun mo‘ljallangan (read-only)** massiv'lar kabi tutadi. Alohida belgilarga `charAt()` metodi bilan murojaat qilish o‘rniga, siz to‘rtburchak qavslardan foydalanishingiz mumkin:
``` js
    let s = "test";
    s.charAt(0) // => "t"
    s[1]        // => "e"
```
Albatta, `typeof` operatori satr'lar uchun hali ham “string” qiymatini qaytaradi va `Array.isArray()` metodiga satr uzatsangiz, u `false` qaytaradi.

Indekslanadigan satr'larning asosiy afzalligi shunchaki `charAt()` chaqiruvlarini ancha ixcham, o‘qishga oson va potensial jihatdan samaraliroq bo‘lgan to‘rtburchak qavslar bilan almashtirishimiz mumkinligidir. Satr'larning o‘zini massiv'lar kabi tutishi, shuningdek, biz ularga umumiy massiv metodlarini qo‘llashimiz mumkinligini ham anglatadi. Masalan:
``` js
    Array.prototype.join.call("JavaScript", " ") // => "J a v a S c r i p t"
```
Yodda tutingki, satr'lar **o‘zgarmas (immutable)** qiymatlardir, shuning uchun ularga massiv sifatida qaralganda, ular faqat o‘qish uchun mo‘ljallangan massiv'lar hisoblanadi. `push()`, `sort()`, `reverse()` va `splice()` kabi massiv'ni joyida o‘zgartiradigan metodlar satr'larda ishlamaydi. Biroq, satr'ni massiv metodi yordamida o‘zgartirishga urinish xatolikka olib kelmaydi: u shunchaki **hech qanday ogohlantirishsiz (silently)** bajarilmay qoladi.
