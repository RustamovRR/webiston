# Unicode Qochish Ketma-ketliklari (Unicode Escape Sequences)

Ba'zi kompyuter qurilmalari va dasturiy ta'minoti Unicode belgilarining to'liq to'plamini ko'rsata olmaydi, kirita olmaydi yoki to'g'ri qayta ishlay olmaydi. Eski texnologiyalardan foydalanadigan dasturchilar va tizimlarni qo'llab-quvvatlash uchun, JavaScript bizga faqat ASCII belgilaridan foydalangan holda Unicode belgilarini yozish imkonini beradigan qochish ketma-ketliklarini belgilaydi. Bu Unicode qochish ketma-ketliklari `\u` belgilari bilan boshlanadi va ulardan keyin to'rtta o'n oltilik raqam (katta yoki kichik A-F harflaridan foydalangan holda) yoki jingalak qavslar ichiga birdan oltitagacha o'n oltilik raqamlardan keyin keladi. Bu Unicode qochish ketma-ketliklari JavaScript satrlarida, muntazam ifoda(regular expression) satrlarida va identifikatorlarda paydo bo'lishi mumkin (lekin til kalit so'zlarida emas). Masalan, `é` belgisi uchun Unicode qochish ketma-ketligi `\u00E9`dir; quyida bu belgini o'z ichiga olgan o'zgaruvchi nomini yozishning uch xil usuli keltirilgan:
``` js
let café = 1;    // Unicode belgisidan foydalanib o'zgaruvchi aniqlash
caf\u00e9        // => 1; qochish ketma-ketligidan foydalanib o'zgaruvchiga murojaat qilish
caf\u{E9}        // => 1; xuddi shu qochish ketma-ketligining boshqa shakli
```

JavaScript’ning dastlabki versiyalari faqat to'rt raqamli qochish ketma-ketligini qo'llab-quvvatlagan. Jingalak qavsli versiya ES6’da 16 bitdan ko'proq talab qiladigan Unicode kod nuqtalari, masalan emoji’larni yaxshiroq qo'llab-quvvatlash uchun kiritilgan:
``` js copy
console.log("\u{1F600}");    // Tabassum emoji’sini chiqaradi
```

Unicode qochish ketma-ketliklari izohlarda ham paydo bo'lishi mumkin, lekin izohlar e'tiborga olinmagani uchun, ular bu kontekstda shunchaki ASCII belgilari sifatida qaraladi va Unicode sifatida talqin qilinmaydi.
