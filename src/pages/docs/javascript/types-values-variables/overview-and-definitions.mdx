# Umumiy ko'rinish va Ta'riflar

JavaScript turlari ikki toifaga bo'linishi mumkin: primitive (sodda) turlar va object (obyekt) turlari. JavaScript'ning primitive turlariga sonlar, matn qatorlari (strings deb nomlanuvchi) va mantiqiy rost qiymatlar (booleans deb nomlanuvchi) kiradi. Ushbu bobning katta qismi JavaScript'dagi raqamli (§3.2) va matn qatori (§3.3) turlarining batafsil tushuntirilishiga bag'ishlangan. Mantiqiy qiymatlar §3.4 bo'limida yoritilgan.

JavaScript'ning maxsus null va undefined qiymatlari primitive qiymatlar hisoblanadi, lekin ular na sonlar, na matn qatorlari, na mantiqiy qiymatlardir. Har bir qiymat odatda o'zining maxsus turining yagona a'zosi sifatida qaraladi. §3.5 bo'limida null va undefined haqida ko'proq ma'lumot berilgan. ES6 Symbol deb nomlanuvchi yangi maxsus maqsadli turni qo'shdi, bu esa til kengaytmalarini aniqlash imkonini beradi.

JavaScript'da son, matn qatori, mantiqiy qiymat, symbol, null yoki undefined bo'lmagan har qanday qiymat - bu obyektdir. Obyekt (ya'ni, object turining a'zosi) - bu xususiyatlar to'plami bo'lib, har bir xususiyat nom va qiymatga ega (bu qiymat primitive yoki boshqa obyekt bo'lishi mumkin). Juda maxsus obyektlardan biri, global obyekt, §3.7 bo'limida yoritilgan, lekin obyektlar haqida yanada umumiy va batafsil ma'lumot 6-bobda keltirilgan.

Oddiy JavaScript obyekti - bu nomlangan qiymatlarning tartiblanmagan to'plamidir. Til shuningdek, array (massiv) deb nomlanuvchi maxsus turdagi obyektni ham belgilaydi, u tartiblangan raqamlangan qiymatlar to'plamini ifodalaydi. JavaScript tili massivlar bilan ishlash uchun maxsus sintaksisni o'z ichiga oladi va massivlar ularni oddiy obyektlardan farqlaydigan ba'zi maxsus xususiyatlarga ega. Massivlar 7-bobning mavzusidir.

Asosiy obyektlar va massivlardan tashqari, JavaScript boshqa foydali obyekt turlarini ham belgilaydi. Set obyekti qiymatlar to'plamini ifodalaydi. Map obyekti kalitlardan qiymatlarga bo'lgan moslashuvni ifodalaydi. Turli "typed array" (tipli massiv) turlari baytlar va boshqa ikkilik ma'lumotlar massivlari ustida amallar bajarishni osonlashtiradi. RegExp turi matnli naqshlarni ifodalaydi va matn qatorlarida murakkab moslashtirish, qidirish va almashtirish amallarini bajarish imkonini beradi. Date turi sana va vaqtlarni ifodalaydi hamda oddiy sana arifmetikasini qo'llab-quvvatlaydi. Error va uning kichik turlari JavaScript kodini bajarishda yuzaga kelishi mumkin bo'lgan xatolarni ifodalaydi. Ushbu turlarning barchasi 11-bobda yoritilgan.

JavaScript statik tillardan shunisi bilan farq qiladiki, funksiyalar va klasslar faqatgina til sintaksisining bir qismi emas: ular JavaScript dasturlari tomonidan boshqarilishi mumkin bo'lgan qiymatlardir. Primitive qiymat bo'lmagan har qanday JavaScript qiymati kabi, funksiyalar va klasslar ham maxsus turdagi obyektlardir. Ular 8 va 9-boblarda batafsil yoritilgan.

JavaScript interpretatori xotirani boshqarish uchun avtomatik axlat yig'ishni (garbage collection) amalga oshiradi. Bu shuni anglatadiki, JavaScript dasturchi odatda obyektlar yoki boshqa qiymatlarning yo'q qilinishi yoki ajratib olinishi haqida qayg'urishi shart emas. Qachonki qiymat boshqa erishib bo'lmaydigan bo'lib qolsa - ya'ni dastur unga murojaat qilishning hech qanday usulga ega bo'lmasa - interpretator u hech qachon qayta ishlatilmasligini biladi va avtomatik ravishda u egallagan xotirani qayta tiklaydi. (JavaScript dasturchilar ba'zida qiymatlar tasodifan erishiladigan - va shu sababli qayta tiklanmaydigan - holatda keragidan ortiq qolmasligini ta'minlash uchun ehtiyot bo'lishlari kerak.)

JavaScript obyektga yo'naltirilgan dasturlash uslubini qo'llab-quvvatlaydi. Sodda qilib aytganda, bu turli xil qiymatlar bilan ishlash uchun global ravishda aniqlangan funksiyalarga ega bo'lish o'rniga, turlarning o'zi qiymatlar bilan ishlash uchun metodlarni belgilashini anglatadi. Masalan, a massivining elementlarini tartiblash uchun, biz a ni sort() funksiyasiga uzatmaymiz. Buning o'rniga, biz a ning sort() metodini chaqiramiz:

a.sort(); // sort(a) ning obyektga yo'naltirilgan versiyasi.

Metodlarni aniqlash 9-bobda yoritilgan. Texnik jihatdan, faqatgina JavaScript obyektlarigina metodlarga ega. Lekin sonlar, matn qatorlari, mantiqiy va symbol qiymatlar xuddi metodlarga ega bo'lgandek ishlaydi. JavaScript'da null va undefined - bu metodlarni chaqirib bo'lmaydigan yagona qiymatlardir.

JavaScript'ning obyekt turlari o'zgaruvchan (mutable) va uning primitive turlari o'zgarmas (immutable) dir. O'zgaruvchan turdagi qiymat o'zgarishi mumkin: JavaScript dasturi obyekt xususiyatlari va massiv elementlarining qiymatlarini o'zgartirishi mumkin. Sonlar, mantiqiy qiymatlar, symbollar, null va undefined o'zgarmasdir - masalan, sonning qiymatini o'zgartirish haqida gapirish ham mantiqsiz. Matn qatorlarini belgilar massivi sifatida tasavvur qilish mumkin va siz ularni o'zgaruvchan deb o'ylashingiz mumkin. Biroq, JavaScript'da matn qatorlari o'zgarmasdir: siz matn qatorining istalgan indeksidagi matnga kirish huquqiga ega bo'lasiz, lekin JavaScript mavjud matn qatorining matnini o'zgartirish uchun hech qanday usul taqdim etmaydi. O'zgaruvchan va o'zgarmas qiymatlar o'rtasidagi farqlar §3.8 bo'limida yanada chuqurroq o'rganilgan.

JavaScript qiymatlarni bir turdan boshqasiga erkin ravishda o'zgartiradi. Masalan, agar dastur matn qatorini kutayotgan bo'lsa va siz unga son bersangiz, u avtomatik ravishda sonni matn qatoriga o'zgartirib beradi. Va agar siz mantiqiy qiymat kutilayotgan joyda mantiqiy bo'lmagan qiymatdan foydalansangiz, JavaScript tegishli ravishda o'zgartirishni amalga oshiradi. Qiymatni o'zgartirish qoidalari §3.9 bo'limida tushuntirilgan. JavaScript'ning erkin qiymat o'zgartirish qoidalari uning tenglik ta'rifiga ta'sir qiladi va == tenglik operatori §3.9.1 bo'limida tasvirlanganidek tur o'zgartirishlarini bajaradi. (Amalda, == tenglik operatori o'rniga === qat'iy tenglik operatori tavsiya etiladi, u hech qanday tur o'zgartirishlarini bajarmaydi. Ikkala operator haqida ko'proq ma'lumot olish uchun §4.9.1 ga qarang.)

Konstantalar va o'zgaruvchilar sizning dasturlaringizda qiymatlarga murojaat qilish uchun nomlardan foydalanish imkonini beradi. Konstantalar const bilan, o'zgaruvchilar esa let bilan (yoki eski JavaScript kodida var bilan) e'lon qilinadi. JavaScript konstantalari va o'zgaruvchilari tiplanmagan: e'lonlar qanday turdagi qiymatlar tayinlanishini belgilamaydi. O'zgaruvchilarni e'lon qilish va tayinlash §3.10 bo'limida yoritilgan.

Bu uzun kirish qismidan ko'rib turganingizdek, bu keng qamrovli bob bo'lib, JavaScript'da ma'lumotlarning qanday ifodalanishi va boshqarilishi haqidagi ko'plab fundamental tafsilotlarni tushuntiradi. Biz JavaScript sonlari va matni tafsilotlarini o'rganishdan boshlaymiz.
